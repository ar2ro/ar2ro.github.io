@import

You can inport stylesheets into stylesheets or even inside the <style> tag in the document head. The @import statement must precede all other CSS rules in a style sheet in order to work properly.

Although @import allow us to write more modular CSS (we can have separate stylesheets for layout, typography etc.), it has an impact in site performance as each @import statement requires a new request to the server.

Pseudo-classes

A pseudo-class is a keyword we add to a selector to style a special state of an element. The most common pseudo-classes are:

  • :link: Targets links that have not been visited by the user.
  • :visited: Targets links that have been visited (clicked) by the user.
  • :hover: Targets an element when a user hovers over it.
  • :active: Gets applied when an element is in an active state (while mouse's left button is pressed, and before it's released, hover an element).
  • :focus: It's only aplied to interactive elements like links, buttons and form elements, as soon as the element receive focus (the elements are navigated with the keyboard).

CSS Data Types

CSS properties has a set of values it can accept, those values are categorizes into data types

Some common data types includes: color, integers, numbers, percentages, length (pixels, ems, rems - Usually we'll define integer and number values with a length unit e.g. "90px").

Absolute length units

Pixels

Absolute units - like pixels - are always fixed in relation to each other (they don't scale like relative units do). When we use absolute units, the size we define will always remain the same and will not scale regardless of the browser window and size of the screen.

As pixel densities are increasing across devices and display resolutions vary widely, nowadays, the pixel unit in CSS doesn't really have anything to do with physical screen pixels, so the CSS pixel is actually an abstract reference unit.

The way CSS pixels are displayed has more to do with the pixel density of the viewing device. For example if we give an element a width and height of 200 pixels, when we view the design on a normal screen, it appears 200px*200px, but when we view it on a device that has twice the pixel density, we might notice that the width and height of the element is actually now equivalent to 400px.

If the pixel density of a device is higher from that of a typical display the display rescales the pixel values, providing consistency across different devices, otherwise our elements might appear a lot smaller on displays with high pixel densities.

Relative length units

Relative units - like percentage - are always relatives to another meassurement unit, in the case of percentages, the percentage value is measured relative to a parent element's length.

Properties like width, margin and padding, accept percentages are usually used for layout since they allow fluid sizing of elements relative to their container.

em unit

The em is called a font relative unit because it's calculated based on a parent elements font size. So if we give an element's property a value of 1em, that will be equal to the font-size value of the parent element.

em units have a compounding effect given by its hineritance nature. Example:

main is 2 times the size of its parent (a document default font size is 16px), this makes it 32px. Then <h1> is 1.625 times the size of its parent (main) 1.625*32px = 52px

rem unit

Similar to an em unit but its only relative to the root element (<html>) of the page. Being only relative to the root element, rem units doesn't have the compounding efect ems have.
