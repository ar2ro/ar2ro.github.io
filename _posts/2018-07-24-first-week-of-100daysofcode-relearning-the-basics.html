Yesterday I completed my first week of the #100DaysOfCode challenge and I've been focusing on relearning the very basics of web development, starting from HTML all the way through CSS and JavaScript.

Contrary to what I thought at first, it took me what I think is a lot of time to review the basics. HTML is a topic in which I had (or maybe still have) a lot of weaknesses, maybe because the way I learned it in the first place, years ago I learned it in a rush without digging any deeper than the basics, since then I've been blindly relying on frameworks (Bootstrap), limiting myself to copy-paste code in an effort to make my layouts work.

So I've spent the entire first week of the #100DaysOfCode challenge focused on HTML, trying to balance my inherent desire to rush things up, and also my desire to dig as deep as I can trying to review every bit of information there is on the subject of my learning.

It's ok to not know everything. In fact it's impossible to.

I've found that, at least for now, there are a lot of topics on which I'll have a lack of knowledge/understanding, I just have to create the habit of review the documentation anytime I hit a wall or I'm in doubt. For example, after trying to validate this website on the W3C validator I got some errors I had to investigate:

  1. I was getting some closing </p> tags without their corresponding opening tag, after investigating in the MDN documentation I found that browsers will close p elements if they find some elements like lists among others. I used to think of p elements like block level elements that I could throw anything into.
  2. I placed an address element with my info and inside it, I enclosed my name inside a h1 heading, well, turns out the address element doesn't accept headings nor sectioning content.

My point here is that it's impossible to try to know every specification for every element, attribute etc. What I think is important is one's ability to investigate and analyze when things broke or when in doubt, and make this a habit.

I also created this blog last week, and I've been updating its code as I went through my learning path. I've also been writing annotations of the topics I either don't understand at first or which I think I need to read again and again until they stick to my memory.

This is the resume of my first week on the challenge:

  1. Created this website from scratch (no framework) and have been updating its code to make it more compliant and semantic. I made it pass the W3C validation without errors or warning.
  2. Completed FCC Introduction to Basic HTML and HTML5 and MDN HTML learning path.
  3. Learned how to use codepen to publish and share code snippets.
  4. Learned how to use (at least the very basics) codeanywhere to run complete projects on the cloud.

Next week objectives (minimum):

  1. Go through a series of CSS tutorials.
  2. Complete at least FCC Basic CSS.
  3. Review MDN CSS section.
  4. Refresh Knowledge by going through TheOdinProject HTML and CSS paths.
